# Claude Code + DeepSeek + Obsidian：从零搭建 AI 知识库

&gt; 完整教程：部署环境 → 挂载模型 → 安装 Obsidian → 建立知识库体系

---

## 第一部分：环境部署

### 第一步：安装 Node.js

Claude Code 依赖 Node.js 环境，需要先装它。

**下载地址：** https://nodejs.org/en/download/

安装完成后，打开终端验证：

```bash
node --version
npm --version
```

两条命令都应该返回版本号。

---

### 第二步：安装 Claude Code

在终端执行：

```bash
sudo npm install -g @anthropic-ai/claude-code
```

&gt; 加 `sudo` 是因为全局安装需要写入系统目录。如果提示密码，输入电脑的开机密码即可。
&gt;
&gt; **Windows 用户：** 不要加 `sudo`。以管理员身份打开终端（右键 → 以管理员身份运行），直接执行 `npm install -g @anthropic-ai/claude-code`。

---

### 第三步：验证安装

```bash
claude --version
```

如果显示了版本号（比如 `0.x.x`），说明 Claude Code 安装成功。

---

### 第四步：下载 CC-Switch

CC-Switch 是一个模型网关层，让 Claude Code 可以调用非 Claude 的模型（比如 DeepSeek）。

**下载地址：** https://ccswitch.io/zh/

&gt; 下载后解压到一个固定的目录（比如 `~/Applications/cc-switch/`），后面配置要用。

---

### 第五步：获取 DeepSeek API Key

1. 打开 DeepSeek 开发者平台 → 注册 / 登录
   **网址：** https://platform.deepseek.com/api_keys
2. 进入 API Keys 页面 → 创建一个新的 Key
3. 复制保存好（关掉页面就看不到了）

#### 为什么要选 DeepSeek？

DeepSeek 的 API 价格很低，能力也够用。一般充 10 块钱可以用很久很久。如果刚开始想简单试试，先充一点点就行。

---

### 第六步：把 API Key 关联到 CC-Switch

1. 启动 CC-Switch（双击或终端运行）
2. 在配置界面里：
   - 添加一个新的模型提供商 → 选 DeepSeek
   - 粘贴刚才复制的 API Key
   - 选择模型（如 DeepSeek V4 Pro）
3. 保存配置

---

### 第七步：测试

确认 CC-Switch 已启动，然后终端运行 `claude` 启动 Claude Code，随便问一个问题（比如"用中文说一句话"）。

- 如果能正常回复 → ✅ 全部通了
- 如果卡住或报错 → 检查 CC-Switch 里的 API Key 和模型配置

&gt; 注意：直接运行 `claude` 只是进入了界面，**不能**验证 DeepSeek API 是否生效。必须完成 CC-Switch 配置后再测试。

## 第三部分：建立知识库体系

### 3.1 笔记的沉淀分区（PARA 体系）

#### PARA 是什么

PARA 是 Tiago Forte 提出的知识管理方法，把笔记分成四个区：

| 分区 | 放什么 | 例子 |
|------|--------|------|
| **1. Projects（项目）** | 有明确目标、有截止日期的短期任务 | 视频脚本、DeepSeek 教程 |
| **2. Areas（领域）** | 长期关注的、没有终点的责任范围 | 健身、滑雪、工作职责 |
| **3. Resources（资源）** | 感兴趣的主题、参考资料 | 电影推荐、书单、音乐收藏 |
| **4. Archives（归档）** | 已完成或不再活跃的内容 | 已发布的教程、过去的项目 |

**判断方法：** 问自己——这份笔记我半年后还要不要改它？
- 要改 → Areas（需要持续维护）
- 不改 → Resources（备查即可）

#### PARA 的两个核心好处

**① 体系化、结构化、可视化你的所有沉淀内容**

没有 PARA 之前：笔记散落在各处，记了找不到，找到也不记得在哪。
有了 PARA 之后：
- 健身的一定在 `2. Areas/健身/`
- 教程的一定在 `3. Resources/教程/`
- 在做的项目在 `1. Projects/`
- 做完了就丢进 `4. Archives/`

每个笔记只有一个归宿，你不需要纠结"这个该放哪"。

**② 应对大模型对话容量有限的问题**

大模型都有上下文窗口限制。GPT-4 128k、Claude 200k——看起来很大，但聊几轮就满了。你关了对话，下次又得从头开始。

解决办法是什么？**把信息沉淀到笔记里，而不是留在对话里。**

你告诉 AI 的东西，AI 直接写成文件存在你的硬盘上。下次你再问，从文件里读，不需要重新说一遍。这就是 PARA + AI 的价值——**你的记忆不会因为对话关闭而丢失。**

#### 本教程的实际例子

```
1. Projects/
   └── Obsidian 体系介绍视频/

2. Areas/
   ├── 健身/
   │   ├── 健身装备清单
   │   ├── 健身减脂方案
   │   └── 训练日志/
   └── 滑雪/
       ├── 滑雪装备清单
       └── 滑雪教练笔记

3. Resources/
   ├── 影视/
   │   └── 侯孝贤电影入坑指南
   ├── 图书/
   ├── 音乐/
   └── Obsidian 部署指南/
       ├── 第一期教程
       └── 第二期教程
```

#### Claudian AI 能帮你做什么

体系建好了，但很多人坚持不下去的原因是：**记笔记本身就有阻力。** 打开 Obsidian、找到正确的文件夹、新建文件、想标题、排版……这些步骤本身就在消耗你的意志力。

Claudian 做的事——把 PARA 体系和 AI 结合起来，让你只需要**说话**。

**① 帮你记录笔记**

你说，AI 记，你不需要碰键盘：

```
你："帮我记一下今天练了背，高位下拉 57.5kg，4组8个"
AI：→ 写入 2. Areas/健身/训练日志/2026-06-22.md
```

**② 根据你的指示，在合理的分区建设合理的笔记**

AI 不只是帮你写——它知道东西该放哪：

```
你："帮我记一下滑雪教练教的东西"
AI：→ 新建 2. Areas/滑雪/滑雪教练笔记.md

你："帮我存一份 DeepSeek 的安装教程"
AI：→ 新建 3. Resources/Obsidian 部署指南/第一期教程.md
```

不需要你告诉 AI 放哪个文件夹。PARA 结构已经在硬盘里了，AI 自动匹配。

### 3.2 知识图谱与笔记关联

PARA 解决了"笔记放哪"的问题。但笔记之间还有一个更重要的维度——**关联**。

#### 双链（Wiki-link）

在 Obsidian 里，用 `[[笔记名]]` 就能链接到另一篇笔记：

```
[[健身减脂方案]]
[[第二期 Obsidian 部署教程]]
```

这看起来只是超链接，但它产生了一个效果：**你的笔记不再是孤立的文件，它们开始产生连接。**

#### 图谱视图（Graph View）

当笔记之间有了链接，打开左侧的图谱视图，你会看到一张不断延伸的知识网络。

**复利效应：**

| 阶段 | 图谱长什么样 | 价值 |
|------|-------------|------|
| 刚开始 | 几个孤立的点 | 看起来没什么 |
| 记了 50 篇 | 点之间有了线 | 开始能看到主题之间的关联 |
| 记了 200 篇 | 密集的网状结构 | 整个知识体系一目了然 |
| 持续积累 | 不断生长 | 覆盖范围越来越大，回顾价值越来越高 |

笔记越多，图越密，你能回顾和调用的范围就越广。这就是「知识复利」。

### 3.3 插件与技能结合

Obsidian 本身有丰富的插件生态，同时这个体系还能和 AI Skill 联动。

#### 推荐插件

| 插件 | 作用 | 应用场景 |
|------|------|---------|
| **Excalidraw** | 手绘风格画图/脑图 | 画流程图、思维导图、概念图 |
| **Templater** | 高级模板引擎 | 一键插入预设模板，保持笔记格式统一 |
| **Tasks** | 待办管理 | 在笔记里直接创建和跟踪任务 |
| **Dataview** | 把笔记当数据库查询 | TASK.md 看板就是用它在跑 |
| **Obsidian Git** | 自动备份到 GitHub | 给所有笔记加了一层保险 |
| **Tag Wrangler** | 批量管理标签 | 标签多了以后统一整理 |

#### 与 AI Skill 结合

这套体系搭好之后，AI 可以真正插手管理你的笔记：

| Skill | 作用 | 配合知识库的效果 |
|-------|------|----------------|
| **stop-slop** | 去 AI 味润色 | 写好的笔记一键变人话 |
| **brainstorming** | 设计前的需求梳理 | 对项目笔记进行结构化的需求分析 |
| **agent-browser** | 浏览器自动化 | 直接从网页抓取内容写入笔记 |

**关键：** 因为你的笔记是纯本地文件、按 PARA 结构组织好了，AI Skill 才能精准地读写它们。如果笔记是一团乱麻，AI 也帮不了你。

## 第四部分：这套体系的核心优势

### 1. 笔记存在你的硬盘里，不会丢

其他笔记软件（Notion、飞书、语雀）：数据在别人的服务器上。哪天服务挂了、账号被封了、公司倒闭了——笔记就没了。

Obsidian：纯本地 Markdown 文件。你电脑上的 `.md` 文件，用任何文本编辑器都能打开。**你的笔记永远属于你。**

### 2. 不需要维护结构

传统知识管理最大的问题：花大量时间分类、打标签、整理文件夹。

有了 PARA，你知道：
- 在做的事 → `1. Projects/`
- 持续关注的事 → `2. Areas/`
- 感兴趣的资料 → `3. Resources/`
- 做完的 → `4. Archives/`

每个笔记只有一个归宿，不需要纠结放哪。

### 3. 笔记之间自动产生关联

其他笔记软件：笔记是孤立的，找东西靠搜索。
Obsidian：用 `[[双链]]` 把笔记连起来。笔记越多，网络越密。打开图谱视图，能看到一张属于你的知识网络。

### 4. AI 可以直接操作你的笔记

因为笔记是纯本地 Markdown 文件，AI 能直接读写它们。

- "帮我记一下今天练背的重量" → AI 自动写到 `Areas/健身/训练日志/`
- "查一下我之前的减脂方案" → AI 直接调出内容
- "帮我把这份笔记整理一下" → AI 直接修改文件

**这是 Obsidian 对比其他笔记软件最根本的优势——你的数据是你的，AI 只是帮你管理的工具。**
